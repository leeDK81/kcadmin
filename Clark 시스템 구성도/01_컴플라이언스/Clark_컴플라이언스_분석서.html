2-1. 본인신용정보관리업 허가 요건 (신용정보법 제22조의9)

📋 허가 유지 의무

  • 자기자본 요건 유지 (5억 원 이상)
  • 물적 설비 (시스템, 보안) 기준 유지
  • 개인신용정보 관리 기준 준수
  • 금융위원회 보고 / 감독 수인 의무

🔄 데이터 전송 요구권 행사 절차 (제33조의2)

  • 정보주체의 명시적 전송 요구 필요
  • 전송 요구 시 포함 사항: 개인신용정보 종류 / 수신기관(Clark) / 전송 주기 및 기간
  • 전송 요구 철회 방법 안내 의무

⚠️ 수집·이용 제한 원칙 (제32조)

  • 수집 목적 범위 내 이용 원칙
  • 목적 외 이용 시 별도 동의 필요
  • 제3자 제공 시: 정보주체 동의 + 제공 목적·항목·기간 명시
2-2. 마이데이터 API 연동 요건 (금융보안원 표준)

🔗 API 연동 표준

  • 마이데이터 표준 API 사용 의무 (스크래핑 금지)
  • API 상호 인증: mTLS 적용 필수
  • OAuth 2.0 기반 사용자 동의 토큰
  • TLS 1.2 이상 암호화 필수 (권고: TLS 1.3)

📦 데이터 보관 제한

  • 정보주체 동의 기간 동안만 보관
  • 동의 철회 시 즉시 삭제 (법정 보존 기간 예외)
  • 마이데이터 원본과 가공 데이터 분리 관리

💡 6. 혁신금융서비스 지정 제도 검토

혁신금융서비스란?

금융위원회가 신기술 기반 금융 서비스에 대해 기존 규제 적용을 일시 면제하고 테스트 기간을 부여하는 제도 (금융혁신지원특별법)

지정 기간: 최초 2년, 최대 4년 연장 가능  |  지정 후 의무: 테스트베드 운영 결과 보고, 금융위 모니터링 수인

Clark 서비스의 혁신금융 신청 검토 결과
서비스기존 규제 충돌혁신금융 신청 검토
마이데이터 보험 연동없음 (기존 마이데이터 허가 범위)✅ 불필요
건강 데이터 + 보험 연계 분석개인정보 민감정보 결합 — 별도 동의로 해소✅ 불필요
KC카드 기반 보장 현황 분석 AI없음 (더파트너스 GA사 허가 범위)✅ 불필요
상담 신청 시 원수사/GA DB 전달없음 (기존 사업에서 검증된 구조)✅ 불필요

✅ 결론 — Clark 서비스는 혁신금융서비스 신청 불필요

아이지넷(마이데이터 허가 범위 내 운영) + 더파트너스(GA사 허가 범위 내 AI 분석) 구조로 각자 기존 허가 범위 안에서 서비스 가능

📋 7. 전자금융거래법 / 금융소비자보호법

💳 전자금융거래법

  • Clark이 직접 자금 이동/결제 처리하지 않는 경우 전자금융업 등록 불필요
  • 본인인증: 공인된 본인확인기관 서비스 이용 (PASS, 카카오 등)
  • 앱 보안 요건: 금융보안원 전자금융 보안 기준 준수 권고

🛡️ 금융소비자보호법 주요 원칙

  • 적합성 원칙: 금융상품 권유 시 소비자 재무 상황·위험 성향 파악 의무
  • 적정성 원칙: 소비자가 요청해도 부적합 상품 판매 제한
  • 설명 의무: 금융상품 주요 내용, 위험, 수수료 설명
  • 불공정 영업 금지: 소비자에 불리한 계약 체결 강요 금지

✅ Clark AI 서비스 적용 — 규제 안전 범위

AI가 금융상품 추천 기능을 탑재할 경우 금융소비자보호법상 금융상품판매업자 등록이 필요할 수 있으나,
Clark은 보장 현황 분석·공백 안내 수준으로 한정 + 더파트너스(GA사)가 운영 → 규제상 문제없음

11-3. 핵심 해소 근거

✅ 아이지넷 → 더파트너스 = 업무 위탁

  • 제3자 제공이 아님 → 신정원 유권 해석 불필요
  • 더파트너스는 아이지넷 지시 하에 AI 분석만 수행하는 수탁자
  • 위탁 계약서 + 처리방침 공개로 해결

✅ GA / 원수사 고객 DB 전달 = 기존 검증 구조

  • 아이지넷 기존 플랫폼에서 동일하게 운영 중
  • 금감원 유권 해석 불필요
  • 상담 신청 버튼 → 동의 후 전달하는 기존 프로세스
11-4. 사업 착수 전 처리 필요 항목 (내부 절차)
1
아이지넷-더파트너스 업무 위탁 계약서 담당: 법무팀  |  난이도: 낮음 (표준 위탁 계약서 형태)
2
개인정보처리방침 업데이트 담당: 법무팀  |  더파트너스 위탁 현황 + CI 보관 목적·기간 명시
3
AI 면책 고지 문구 법무 확정 담당: 법무팀  |  "보험 권유 아님, 전문가 상담 권고" 문구 확정
4
건강 데이터 별도 동의 화면 설계 담당: UX팀  |  더파트너스 명시, 민감정보 고지, 거부 가능 명시
