# Agent C — 전자금융감독규정 국내설치의무

## 조사방법 한계
law.go.kr 직접 텍스트 추출 실패, 위키문헌(ko.wikisource.org) 미러본(전자금융감독규정 제2025-4호)으로 원문 확인. 2차 출처 다수 교차.

## 1. 정확한 조번호·원문·최신 개정일
**정정: "제14조의2 제8항"으로 알려진 것은 구버전(2018-36호) 항번호. 2025-4호 최신판 기준 국내설치 문구는 제7항.**

전자금융감독규정 제14조의2(클라우드컴퓨팅서비스 이용절차 등) 제7항 원문(2025-4호):
> "제1항의 절차를 거친 클라우드컴퓨팅서비스 제공자의 정보처리시스템이 위치한 전산실에 대해서는 제11조제1호 및 제2호, 제15조제1항제5호를 적용하지 아니한다. 다만, 금융회사 또는 전자금융업자(전자금융거래의 안전성 및 신뢰성에 중대한 영향을 미치지 않는 외국금융회사의 국내지점, 제50조의2에 따른 국외 사이버몰을 위한 전자지급결제대행업자는 제외한다)가 고유식별정보 또는 개인신용정보를 클라우드컴퓨팅서비스를 통하여 처리하는 경우에는 제11조제2호를 적용하고, 해당 정보처리시스템을 국내에 설치하여야 한다."

- 적용대상: 금융회사 또는 전자금융업자
- 예외: 외국금융회사 국내지점(안전성에 중대영향 없는 경우), 국외 사이버몰 대상 전자지급결제대행업자 — 이 둘은 개인신용정보 처리해도 국내설치의무 자체 없음.
- 최신본: 2025-4호. 2025.5.22 개정(클라우드 이용절차 간소화, 망분리 특례 확대)이 있었으나 제7항 문구 자체 변경 여부는 확인 안 됨. 2026.1.20~2.9 시행세칙 개정 사전예고(SaaS 확대) 진행중이나 이 조항 개정여부 확인 안 됨.

## 2. 마이데이터 사업자(본인신용정보관리업자)에게 적용되는가
**결론: 원칙적으로 적용 안 됨 — 판단기준은 라이선스 종류가 아니라 "전자금융업무 수행 여부".**

- 문언상 적용대상은 "금융회사 또는 전자금융업자"(전자금융거래법 개념). 본인신용정보관리업 허가는 전자금융거래법상 "전자금융업자"(직불/선불전자지급수단발행업, 전자지급결제대행업 등 열거업종) 범주에 자동 포함 안 됨.
- **CaseNote 금융위·금감원 법령해석**: "전자금융업무를 하지 않는 여신전문금융회사 등은 제14조의2에 따른 클라우드컴퓨팅서비스 이용절차가 적용되지 않는다" — 판단기준이 실제 전자금융업무 수행여부임을 확인. 마이데이터 사업자 특정 사례 명시는 없음(확인 안됨)이나 동일논리 적용시, **본인신용정보관리업만 영위하고 전자금융업 겸영 안 하면 제14조의2(국내설치 포함) 직접 적용 안 될 가능성 높음**.
- 신용정보업감독규정에는 침입차단·침입탐지, 백신, 접근권한 최소화, 위탁시 업무장소/전산설비 분리 등 일반 보안조치 의무는 있으나 "국내설치"에 상응하는 명문 조항은 확인 안 됨.
- **실무적 함의: 아이지넷이 전자금융업을 별도 겸영하는지가 결정적 변수.**

## 3. "개인신용정보처리시스템"에 RAG 벡터DB/AI엔진 포함되는가
**결론: 시스템 유형이 아니라 처리되는 데이터 성격이 기준 — 개인신용정보 처리시 RAG/AI엔진도 포함 가능성 높음.**

- 전자금융감독규정 제2조 "정보처리시스템" 정의는 포괄적(하드웨어+소프트웨어+관련장비), 기술유형 특정 안 함.
- 제14조의2 제7항 문언도 "고유식별정보 또는 개인신용정보를 클라우드컴퓨팅서비스를 통하여 처리하는 경우"가 기준 — 데이터 성격 기준.
- **관련 유권해석(CaseNote)**: 금융회사가 AI서비스로 뉴스/재무정보 요약, AI챗봇 고객응대하는 외부 SaaS(클라우드기반 AI) 이용 사안에서, 금융위·금감원은 "클라우드컴퓨팅서비스"에 해당하므로 제14조의2 이용절차 준수 필요하다고 결론 — AI/RAG형 서비스도 일반클라우드와 동일규제 대상됨을 시사.
- 금융위 생성형AI 활용지원방안(2025): 상용AI(외부클라우드)는 개인신용정보 미활용 용도로 제한권고, 개인정보 다루려면 오픈소스AI를 내부망 직접설치 권장 — "개인신용정보 다루는 AI/RAG=국내(내부망) 설치대상"이라는 정책방향과 일치.
- 2025.8.6 해설서 개정판에 클라우드/생성형AI/BNPL 반영됐다는 언급 있으나 세부문구(RAG를 명시적으로 분류했는지)는 확인 안됨.

## 4. "국내"의 의미
**결론: 사업자 국적 무관, 물리적 대한민국 영토 내 설치면 충족. NCP 한국, AWS 서울리전 모두 요건 충족.**

- "전자금융업자는 국내에 이미 전산센터를 갖춘 해외 클라우드 사업자의 서비스도 이용 가능" — 국내 사업자 소유/운영 요구 아니라 물리적 소재지 기준.
- AWS 서울리전은 K-ISMS 인증(2023.12.16~2026.12.15) 등 국내규제대응체계 보유.
- "국내 사업자 시설이어야 한다"는 더 엄격한 요건은 확인 안 됨 — 존재하지 않는 것으로 판단.
- **단, 이는 제14조의2가 적용되는 주체(금융회사·전자금융업자)에 한정된 이야기 — 마이데이터 사업자에게 조항 자체가 적용되는지는 2번 항목의 별개 문제.**

## 5. 2025~2026 최신 동향
- 2025-4호 현재 유효, 국내설치 문구(제7항) 유지.
- 2025.5.22 개정: 클라우드 이용계약 신규체결시 3개월내 금감원보고, 연구개발목적 망분리예외 확대 — 국내설치의무 자체 폐지/변경 근거 없음.
- 2025.8.6 해설서 전면개정 — 세부내용 원문 미확보(확인 안됨), 직접 대조 권고.
- 2026.1.20~2.9 시행세칙 개정 사전예고(SaaS 확대) — 국내설치조항 개정여부 확인 안됨.
- 금융위 생성형AI 활용지원방안: 상용AI-개인신용정보 분리권고 정책기조 형성중 — 향후 RAG/AI 국내설치·망분리 규제 정교화 가능성.

## 종합판단(실무 시사점)
1. 조번호 정정: 국내설치 문구는 제14조의2 **제7항**(구버전 "제8항" 인용은 오기 가능성).
2. **이 조항은 전자금융업자·금융회사에게만 직접 적용 — 마이데이터(본인신용정보관리업)만 영위시 자동적용 안 될 가능성 높음.** 아이지넷의 전자금융업 겸영 여부 확인 필요(★ 매우 중요 확인사항).
3. 개인신용정보처리시스템은 데이터성격 기준 — RAG DB/AI엔진도 개인신용정보 다루면 적용대상 가능성 높음(단, 명시적 유권해석은 확인 안됨).
4. NCP 한국·AWS 서울리전 모두 물리적 국내소재로 국내설치요건 자체는 문제없음(조항이 적용된다는 전제하에).
5. 마이데이터 전용사업자면 국내설치의무보다 신용정보법상 위탁규정·안전성확보조치·국외이전규제가 실제 규율근거일 수 있음 — 신용정보업감독규정 원문 직접대조 권고, 명문 국내설치조항은 이번조사로 미발견.
