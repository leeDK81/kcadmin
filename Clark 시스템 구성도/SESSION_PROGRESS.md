## 확정된 핵심 정책

### 데이터 흐름
```
[사용자]
  ↓ 더파트너스 서비스 이용 동의 + 건강 데이터 별도 동의
[Clark App — 아이지넷 NCP]
  ↓ 업무 위탁: 나이·성별·보험내역·건강정보 (CI 절대 미포함)
[더파트너스 KC카드 AI 엔진 — AWS Bedrock 서울 리전]
  ↓ 분석 완료 → 보장 현황 결과 반환 → 임시 데이터 즉시 파기
[Clark App 화면 출력 + 더파트너스(GA사) 제공 서비스 명시]
  ↓ 사용자가 상담 신청 → 동의 후 원수사/GA에 고객 DB 전달
```

### CI 정책
- 아이지넷 내부 DB에서만 단방향 해시 보관
- 목적: 탈퇴 후 재가입 동일인 확인
- 보관: 탈퇴 후 1년 후 자동 파기
- 외부 전송: 더파트너스 포함 모든 외부 절대 금지

### LLM 정책
- AWS Bedrock (서울 리전) — 더파트너스 기존 AWS 인프라 활용
- 국외 이전 없음

## 다음 세션에서 이어할 수 있는 작업

컴플라이언스·법적 검토 완료. 아래 작업은 사업 필요에 따라 선택적으로 진행:

- **API 설계**: 아이지넷↔더파트너스 인터페이스 스펙, 마이데이터 표준 API, 프롬에이지/쿠콘 연동
- **화면 설계**: 건강 데이터 동의 화면, 보험 진단 결과 화면, 채팅 UI 화면, 상담 신청 흐름
- **데이터 설계**: DB 스키마, 개인정보 처리 항목, 동의 이력 관리, 채팅 대화 로그 보관 정책
- **보안 설계**: 인증 체계 상세, ISMS 요건, CI 암호화 구현
- **가드레일 설계**: 채팅 질문 분류 모델, 차단 로직 상세 스펙
