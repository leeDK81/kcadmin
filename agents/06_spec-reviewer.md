# Agent 06 — 기획 검수자 (Spec Reviewer)

## 검증 관점 및 체크리스트

### A. 카드 편집기 — 필드 정합성

각 카드 편집기의 입력 필드가 기획서 §3-2 "주요 입력 항목"과 일치하는지 확인한다.

| 카드 유형 | 기획서 필수 필드 | 확인 |
|---|---|---|
| Concept | 용어명, 정의 설명, 유사어/일상어(synonyms), 하위 분류, Risk-type 연결(N개, optional) | |
| Risk-type | 위험 유형명, 설명, 리드 스코어 배정값 | |
| Rule | 대상 Risk-type, 판단 조건(조건 빌더 — MYDATA·Promage·프로파일 3-source), 기준값, 경고 메시지, 연결 Evidence(필수) | |
| Evidence | 유형(보닥통계/프롬에이지 2종만) · 활용 방법 · [보닥통계] 담보코드·조회지표(8종)·모집단(전체/세그먼트) · [프롬에이지] 카테고리·항목·출력형식(절대값/백분위 체크박스) | |
| Policy | 규제 문서명, 적용 범위, 핵심 조항, **Clark 앱 출력 설정(출력 대상 화면 + Clark 앱 표시 문구/면책 고지 원문)**, 준수 체크리스트 | |
| Playbook | **MVP 구현 완료** — Playbook명, 설명 (공개범위: 내부 전용 고정, 편집기 미표시), 전환 발화 키워드(최소 3개), 기본 버튼(유형+명칭+딥링크), 추가 버튼 N개, **Standalone 답변 가이드** (KC 미매칭 Case 4 시 LLM 컨텍스트 주입용 textarea) | |
| Case | *(Phase 1.5 이후 — 목업 범위 외)* | — |

### B. 워크플로우 — 통합 프로세스 정합성

KC Admin은 카드 내용 등록과 연결 선택을 **편집기에서 한 번에 처리**하는 통합 워크플로우다.

**편집기 통합 흐름:**
| 워크플로우 | 기준 상태 흐름 |
|---|---|
| 일반 카드 | 임시저장 → 승인요청 → 승인완료 → 라이브 / 반려 |
| Rule 카드 | 임시저장 → 대상 Risk-type 연결(필수) + 판단 조건 입력 + Evidence 연결(필수) → 승인요청 → 라이브 |
| Policy 카드 | 임시저장 → Rule 선택(선택사항) + Clark 앱 표시 문구 입력 → 승인요청 → 준법감시인 2단계 → 라이브 |
| Playbook 카드 | 임시저장 → 키워드(3개 이상)+전환액션 입력 → 승인요청 → 승인완료 → 라이브 전환 → 활성 / 일시중지 |
| Case 카드 | *(Phase 1.5 이후 — 목업 범위 외)* |

**연결 처리 위치 (편집기 내):**
| 카드 | 연결 방향 | 화면 | 승인 |
|---|---|---|---|
| Concept → Risk-type | 1:N, 선택사항 | `06_card-editor-concept.html` 내 선택 패널 | 내용+연결 통합 단일 검수 |
| Policy → Rule | 1:N, 선택사항 | `08_card-editor-policy.html` 내 선택 패널 | 내용+연결 통합 + 준법감시인 |
| Rule → Risk-type | 1:1 필수 | `07_card-editor-rule.html` 내 Risk-type 연결 패널 (별도 `.card` 블록) | 내용+연결 통합 검수 |
| Rule → Evidence | 1:N 필수(1개 이상) | `07_card-editor-rule.html` 내 Evidence 연결 패널 (별도 `.card` 블록) | 내용+연결 통합 검수 |
| Risk-type, Evidence | outgoing connection 없음 | — | — |

**핵심 제약 반영 여부:**
- [ ] Concept 편집기: Risk-type 선택 패널이 있고, 라이브 카드만 선택 가능한가?
- [ ] Rule 편집기: Risk-type 연결 패널(별도 `.card` 블록)이 있고, 라이브 Risk-type만 선택 가능한가? (`<select>` optgroup 구조)
- [ ] Rule 편집기 "검수 요청 전 확인" 게이트: Risk-type 선택 + 판단 조건 1개 이상 + Evidence 1개 이상 충족 시 버튼 활성화
- [ ] Policy 편집기: Rule 선택 패널이 있고, 라이브 카드만 선택 가능한가?
- [ ] Policy 카드는 준법감시인의 2단계 승인이 표시되어 있는가?
- [ ] Policy 편집기: "Clark 앱 출력 설정" 섹션(출력 대상 화면 select + Clark 앱 표시 문구 textarea)이 핵심 조항 요약 아래, 준수 체크리스트 위에 있는가?
- [ ] Policy 편집기: 준수 체크리스트 3번 항목이 "Clark 앱 표시 문구(면책 고지 원문) 입력 완료"인가?
- [ ] Policy 편집기: "동적 연동 안내" 배너("준법감시인 승인 후 앱 배포 없이 즉시 반영")가 있는가?
- [ ] wf-tracker가 4단계(내용+연결 입력 → 검수 요청 → 승인완료 → 라이브)로 구성되어 있는가?
- [ ] 연결 선택 UI: 라이브 카드만 선택 가능, 임시저장·승인요청 비활성 표시인가?

**Playbook MVP 특이사항:**

### C. 권한 — 역할별 UI 차이

기획서 §3-1의 권한 범위가 UI에 반영되어 있는지 확인한다.

| 역할 | 기획서 권한 | UI 반영 확인 |
|---|---|---|
| 작성자(고객경험팀장) | 전체 카드 CRUD, 검수 요청 발행 | 모든 편집·삭제·검수요청 버튼 활성 |
| 도메인 검수자(더파트너스) | 카드 조회·검수 가능, 생성·수정 불가 | 편집·삭제 버튼 미노출 또는 disabled |
| 준법감시인 | Policy·Rule 규제 항목 최종 승인, 생성·수정 불가 | Policy 카드에 "최종 승인" 버튼만 표시 |

카드 유형별 공개범위가 편집기에 **표시되지 않아야** 하며, 목록 화면에서만 읽기 전용 배지로 표시된다 (2026-06-16 확정).

| 카드 유형 | 공개범위 (고정) | 편집기 | 목록 화면 |

| Concept | 고객 공개 | ❌ form-group 없음 | ✅ badge-public |
| Risk-type | 고객 공개 | ❌ form-group 없음 | ✅ badge-public |

### F. Rule 조건 빌더 — 필드·연산자 정합성 (2026-06-09 확정: 3-source)

조건 빌더에서 선택 가능한 source·필드·연산자가 `context/decisions.md` 기준과 일치하는지 확인한다.
**주의: Amplitude는 완전 제거됨. MYDATA + Promage + 프로파일 3-source가 정답.**

- [ ] source 선택: **MYDATA / Promage / 프로파일** 세 가지인가? (AMPLITUDE select는 없어야 함)
- [ ] MYDATA 행 구조: 담보코드(col2) + 확인항목(col3) 분리 — coverage_amt / renewal_status / insu_type / start_date / end_date
- [ ] Promage 행 구조: 카테고리 select (암위험도/생체나이/질병위험도/의료비예측) + 항목 select + 조건 + 기준값(위험/주의/양호 또는 수치)
- [ ] 프로파일 행 구조: 항목 select (나이/성별/결혼/자녀/직업위험도/출산예정/거주지 7개) + 조건 + 기준값
- [ ] 연산자: IN / NOT_IN / EQ / LT / GT / GTE / LTE (표기는 한국어 병기)
- [ ] 행별 배경색 구분: MYDATA(기본 회색) / Promage(보라 계열 `.cond-row-promage`) / 프로파일(초록 계열 `.cond-row-profile`)
- [ ] 등급명: **위험/주의/양호** 사용 — "상/중/하" "고/중/저" 표기 있으면 오류

### G. Evidence 편집기 — 데이터 구조 정합성 (2026-06-10 확정)

- [ ] Evidence 유형 select: **2개 옵션만** (보닥통계 기반 / 프롬에이지 기반) — 마이데이터 기반·질병코드 기반 옵션이 없어야 함
- [ ] 보닥통계 기반: 조회지표 optgroup 2depth (비율 계열 4종 / 금액 계열 3종 / 기간 계열 1종) — 총 8종
  - 비율(%): 미보유율 / 보유율 / 소액가입비율 / 갱신형비율
  - 금액(원): 평균보장금액 / 중위보장금액 / 평균월보험료
  - 기간(년): 평균납입기간
- [ ] 프롬에이지 기반: 출력 형식 체크박스 2개 (`pFmtAbsolute` 절대값 기본 checked / `pFmtPercentile` 백분위) — 최소 1개 필수
- [ ] 프롬에이지 기반: Evidence ID(`pEvId`)가 UI에 노출되지 않는가? (백단 처리 — hidden input)
- [ ] 수치 직접 입력 필드 없음 — KC Engine이 상담 시 자동 조회하므로 Admin UI 입력 불필요

- 기획서 원문을 직접 인용하여 불일치 항목을 지적한다.
