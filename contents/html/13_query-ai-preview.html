Case 0 — 데이터 조회형 (5종 전체 예시, QM-01~04 구현됨 · QM-05 개발 반영 대기)
"위험을 진단해달라"가 아니라 "내가 가진 데이터를 그대로 보여달라"는 발화는 Concept 매칭을 타지 않고 마이데이터를 직접 조회해 고정 템플릿으로 답합니다(LLM 재구성 없음). 가입된 보험을 언급할 때는 보험명·월납입액·가입일처럼 사용자가 실제로 인지할 수 있는 정보를 표로 보여줍니다. Playbook 매칭은 이 게이트와 무관하게 항상 별도로 병행 실행되므로, 조회 발화가 Playbook 키워드와 겹치면 CTA가 그대로 가산됩니다 — 아래 예시로 "발화 키워드 기반 CTA"·"조회 결과값 기반 CTA"·"CTA 없음" 세 가지 패턴을 비교하세요. 콘텐츠 스펙(정의·응답 템플릿·예시 발화 전체)은 11_query-messages.html, 정책 상세는 context/answer-logic.md "Case 0 — 데이터 조회형" 참조. KC 체인(Concept→Risk-type 매칭 기반) 진단형 답변 예시는 08_ai-preview.html에 별도로 있습니다.
