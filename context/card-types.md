# KC Admin — 카드별 상세 스펙

> **진실원.** 각 카드 유형의 필드 정의, 기능 스펙, 운영 규칙은 이 파일이 기준.
> 변경 시 PO(02)가 이 파일을 즉시 업데이트한다.

## Risk-type 카드

**역할:** Clark AI가 보험 리스크를 분류하는 기준 카드. Concept·Rule 카드가 이 카드를 참조해 KC 체인을 구성.

**중요도 (2026-06-22 확정):**

| 중요도 | 의미 |
|---|---|
| 높음 | 핵심 진단 유형 — 다른 유형과 동시 감지 시 선노출 |
| 보통 | 일반 진단 유형 — 기본값 (미설정 시 자동 적용) |
| 낮음 | 보조 진단 유형 — 상위 유형 노출 후 후순위 안내 |

**다중 감지 우선순위 (2026-06-22 재확정):**

| 순위 | 기준 | 의미 |
|---|---|---|
| 1순위 | 중요도 서열 (높음 > 보통 > 낮음) | 운영자 판단 — 서비스 전략 |
| 2순위 | 선택 조건 충족 개수 | 시스템 자동 계산 — 개인화 |
| 3순위 | 최근 배포순 | 타이브레이크 (가장 최근 라이브 전환일 우선) |

**가중치 완전 제거됨:** 중요도 ×3/×2/×1 가중치 개념 삭제. 높음/보통/낮음 서열만 사용.

## Concept 카드

**역할:** 사용자 질문의 핵심 단어를 동의어로 매칭해 Risk-type 체인으로 연결하는 입구 카드.

- Concept → Risk-type 연결 **필수** (최소 1개). 연결 없으면 KC 체인 미진입.
- Standalone 기능 없음 — 완전 제거됨 (구 기획 흔적 코드 삭제 대상)
- 미매칭 시 Case 3 (RAG→Fallback) 또는 Case 4 (Playbook 감지) 분기

## Evidence 카드

**역할:** Rule 판단 기준값의 근거를 제공하는 공인 통계 데이터 카드. Clark 상담 시 추천 이유 설명에 활용.

**단일 유형: 공인 외부 통계 기반** (code: `external`) — 유형 select 없음.

**편집기 필드 (2026-06-17 확정):**

| 필드 | 필수 | 설명 |
|---|---|---|
| 제목 | ✅ | 핵심 내용 한 줄 요약 |
| 출처 기관 | ✅ | HIRA / NHIS / KOSTAT / FSS / 기타 |
| 보고서명 | ✅ | 인용한 공식 보고서명 |
| 기준 연도 | ✅ | 통계 발행 연도 |
| 지표명 | ✅ | 활용할 통계 지표 명칭 |
| 기준값 | ✅ | 단위 포함 입력 (예: 3,800만원, 68%) |
| 출처 URL | 선택 | 원본 자료 링크 |
| 활용 방법 | ✅ | 어떤 Rule 기준값의 근거인지, 왜 이 수치가 기준인지 |

**폐지 항목:** 보닥통계(stat)·프롬에이지(fromage) 유형 — 완전 제거됨.

## Rule 카드

**역할:** 마이데이터·프롬에이지·프로파일 3개 소스 기반으로 Risk-type 판정 조건을 정의하는 카드. Evidence와 연결해 판단 근거를 갖춤.

### Rule 조건 빌더 3-source (2026-06-08 확정 / 2026-06-22 재확정)

**MYDATA + PROMAGE + 프로파일** — Amplitude 완전 제거.

| source | 화면 표기 | 배경색 클래스 | 기본 required |
|---|---|---|---|
| MYDATA | 마이데이터 | `.cond-row-mydata` (기본 회색) | `true` (필수) |
| PROMAGE | 프롬에이지 | `.cond-row-promage` (보라, #FAF8FD) | `false` (선택) |
| PROFILE | 프로파일 | `.cond-row-profile` (초록, #F4FBF8) | `true` (필수) |

**조건 구분:**

| 구분 | 동작 | 역할 |
|---|---|---|
| 필수 조건 | 모두 충족해야 Rule 발동 (AND) | 리스크 해당 여부 판단 기준선 |
| 선택 조건 | 미충족 시에도 Rule 발동 | 충족 개수가 Risk-type 우선순위 2순위에 활용됨 |

**PROMAGE 미연동 대응:** 프롬에이지 미연동 사용자는 PROMAGE 조건 평가 불가. 따라서 PROMAGE 조건은 선택 조건으로 설정 권장. 운영자가 필수로 변경하는 것은 허용 (의도적 설계).

**조건 빌더 컬럼 구조 (2026-06-26 6열 통일):**

| 열 | 헤더 | 소스별 내용 |
|---|---|---|
| 1 | 소스 | 마이데이터 / 프롬에이지 / 프로파일 select |
| 2 | 담보코드·카테고리·항목 | MYDATA: 담보코드 select / PROMAGE: 위험도 항목 select / PROFILE: 나이·성별·결혼여부·자녀유무·운전여부 select |
| 3 | 조건 | MYDATA: op select(LTE/GTE 등) / PROMAGE: 같음(=) static / PROFILE 나이: 사이(∈) static / PROFILE 나머지: 같음(=) static |
| 4 | 기준값 | MYDATA: 금액 input / PROMAGE: 위험·주의·양호 select / PROFILE 나이: 시작세~종료세 dual input / PROFILE 나머지: 값 select |
| 5 | 필수여부 | 필수 / 선택 radio |
| 6 | (삭제) | 삭제 버튼 |

**PROMAGE 조건 구조 (2026-06-25 단순화 / 2026-06-26 optgroup 적용):**
- **카테고리 단일화:** 암위험도 + 질병위험도 → **위험도** (50개 항목 통합). 생체나이·의료비예측 삭제.
- **op:** EQ only — 위험 / 주의 / 양호 등급 선택
- **편집기 Row:** source select → 항목 select (optgroup 2개) → EQ select → 등급 select
- **optgroup 구분:** `<optgroup label="암위험도">` 23개(canr_ca~smbl_ca) + `<optgroup label="질병위험도">` 27개(고지혈증~췌장염)

**MYDATA Rule 조건:** 담보코드 select + 담보금액(coverage_amt) LTE/GTE/LT/GT/EQ. 필드 드롭다운 없음.

**USER_PROFILE_FIELDS for Rule 조건 (2026-06-26 업데이트):**

| 필드코드 | 레이블 | op | 기준값 | 비고 |
|---|---|---|---|---|
| age | 나이 | BETWEEN | 시작세 ~ 종료세 dual input (예: 40~65세) | 범위 조건 |
| gender | 성별 | EQ | 무관/남/여 select | 무관=전체 적용 |
| married | 결혼여부 | EQ | 무관/기혼/미혼 select | 무관=전체 적용 |
| has_child | 자녀유무 | EQ | 무관/있음/없음 select | 무관=전체 적용 |
| drives | 운전여부 | EQ | 무관/유/무 select | 무관=전체 적용 |

**등급명:** 위험 / 주의 / 양호 — "상/중/하" "고/중/저" 표기 금지.

### 약관 DB 자동 참조 (2026-06-24 정책 변경)

- **폐기:** Rule 카드의 "약관 DB 연동" 체크박스 제거 (2026-06-24)
- **동작:** 약관 DB 확보 시 → Rule 조건 충족 → 담보코드 기준 자동 조회 → Clark 답변에 면책조항 섹션 자동 포함. 플래그 없이 엔진이 자동 처리.
- **약관 DB 관리:** 크롤링 파이프라인이 담당 — KC Admin에서 직접 수정 불가

## Policy 카드 (2026-06-22 필드 재확정)

**역할:** Clark AI 답변의 출력 범위와 형식을 규제하는 카드. Rule 카드와 연결하여 KC 기반 답변 출력을 제어.

**확정 필드 2개:**

| 필드 | 필수 | 설명 |
|---|---|---|
| Policy 이름 (name) | ✅ | 식별용 내부 명칭 |
| Clark 앱 표시 문구 (appDisplayText) | ✅ | 운영자가 직접 작성. 비어있으면 검수 요청 불가. |

**승인 2단계:** 도메인 검수자 → 준법감시인

**삭제된 필드 (코드·문서에서 완전 제거):** 규제 문서명, 적용 범위, 핵심 조항 요약, 출력 대상 화면, 출력 제한 설정, 준수 체크리스트

## Playbook 카드 MVP (2026-06-08 확정 / 2026-06-22 재확정)

**역할:** KC 체인과 병렬로 동작하는 독립 카드. 발화 키워드를 감지하면 CTA 버튼(상담 연결·앱 이동·외부 링크)을 노출하고 GA에 리드 데이터를 전달.

**공개범위:** 내부 전용 고정 (변경 불가). 색상: `--card-playbook: #6B4A9A`.

**편집기 6-card 구조 (2026-06-21 업데이트):**

| 카드 | 내용 |
|---|---|
| Card ① | 기본정보 — Playbook명, 설명 |
| 연결 안내 | Playbook은 다른 카드와 직접 연결 없음 (Phase 1.5+ 검토 중) |
| Card ② | 리드 전환 레이어 — 발화 키워드(최소 3개) + 전환 액션(consult 필수 + 추가 버튼 N개) + Standalone 답변 가이드(선택사항) |
| Card ③ | 리드 스코어링 — Phase 1.5 이관. scoring-inactive/active 토글 구현 |
| Card ④ | 배정 최적화 — Phase 2 잠금 UI |
| Card ⑤ | 액션 — 검수 요청 / 임시저장 |

**버튼 유형 3종:**

| value | 표시 | 역할 |
|---|---|---|
| `consult` | 💬 상담 연결 | GA lead data 전송 트리거. Card ③ 활성화 조건 |
| `deeplink` | 🔗 앱 내 이동 | 딥링크 기반 인앱 이동 |
| `external` | 🌐 외부 링크 | 외부 URL 이동 |

**최소 요건 (2026-06-22 확정):**
- 발화 키워드: 최소 3개 (필수)
- CTA 기본 버튼 (`consult`): 필수
- Standalone 답변 가이드: **선택사항** (비워두면 Clark 기본 안내 문구 적용)
- ~~"최소 20자 필수"~~ — 삭제됨

**Playbook 라이브 전환:** approved 상태에서 캔버스 연결 없이 직접 "라이브 전환" 가능 (독립 체인, KC 카드 연결 불필요).

**상태별 액션 버튼:**

| 상태 | 버튼 |
|---|---|
| draft | 편집 + 삭제 |
| review | 요청 취소 |
| approved | 라이브 전환 + 편집 |
| active | 일시중지 + 편집 |
| paused | 재시작 + 편집 |

**참조 파일:** `13_answer-logic.html` (`14_answer-logic-guide.html`, `17_playbook-answer-guide.html`은 삭제된 파일 — 사용 금지)

## 프롬에이지 연동 설계 (2026-06-17 재확정)

Clark 앱 Step 5 → "더 정밀한 진단" → Step 6 건강정보 연동 동의 → Step 7 고도화 진단.

**프롬에이지는 Evidence 카드 불필요 — 상담 시 자동 전달:**
- 마이데이터·프롬에이지·프로파일은 상담 시 KC 엔진에 실시간 자동 전달
- Rule 조건 판단에 직접 활용 (Evidence 카드 별도 등록 없음)
- ~~EV.FROMAGE.*~~ 폐기됨 (2026-06-17)

**처리 원칙:**
- 원본 수치 → 보닥 서버(국내)에만 저장
- Clark 출력 → 위험/주의/양호 3단계 등급명만 (수치·percentile 노출 금지)

## LLM 기본 응답 가이드라인 (시스템 설정 — 2026-06-19 확정)

**위치:** `18_system-settings.html` — KC 카드 체인과 독립. 저장 즉시 Clark AI에 반영. 상태 개념 없음 (항상 라이브).

**설정 필드:**

| 필드 | 필수 | 설명 |
|---|---|---|

| 마무리 안내 문구 | 선택 | 모든 Fallback 답변 끝에 자동 추가되는 전문가 연결 유도 문구 |
