# KC Admin — Concept·Playbook 감지 정책

> **진실원.** 사용자 질의 → Concept·Playbook 매칭 방식과 결과 처리 규칙.
> 변경 시 PO(02)가 이 파일을 즉시 업데이트한다.
> 구현 세부(기준값·모델 선택)는 개발팀이 결정 후 이 파일에 반영한다.

---

## 개요

사용자 질의가 들어오면 **Concept 매칭**과 **Playbook 매칭**을 동시에 실행한다.
둘은 역할이 다르므로 독립적으로 결과를 낸다.

| 카드 | 역할 | 결과 |
|---|---|---|
| Concept | KC 답변 체인 진입 트리거 | KC 구조화 답변 생성 |
| Playbook | CTA 노출 트리거 | 버튼·안내 레이어 추가 |

---

## 매칭 방식 — 임베딩 의미 유사도

두 카드 모두 **임베딩 벡터 유사도**로 매칭한다.
키워드 완전 일치 방식을 쓰지 않는다.

**이유:**
- 운영자가 등록하지 않은 표현(동의어·구어체·오탈자)도 의미가 같으면 잡아야 한다.
- "암"만 등록해도 "악성종양", "종양 진단", "cancer" 같은 표현이 매칭되어야 한다.
- Concept 동의어·예문과 Playbook 예상질의 모두 문장·단어 혼용이 가능하므로 동일 엔진으로 처리한다.

---

## 인덱싱 대상

### Concept 카드
- **동의어·예문 필드 전체** (운영자가 입력한 단어·구·문장 전부)
- 카드 ID와 함께 인덱싱 → 매칭 시 어느 Concept인지 식별

### Playbook 카드
- **예상 질의 필드 전체** (운영자가 입력한 문장들)
- 카드 ID와 함께 인덱싱 → 매칭 시 어느 Playbook인지 식별

> Concept의 "용어명(대표명)"은 인덱싱 필수. 동의어 미등록 카드도 대표명으로는 매칭되어야 함.

---

## 실행 구조

```
[사용자 질의 입력]
        │
        ├──────────────────────────────┐
        ▼                              ▼
[Concept 벡터 검색]          [Playbook 벡터 검색]
  (동시 병렬 실행)              (동시 병렬 실행)
        │                              │
        ▼                              ▼
 유사도 최고점 1개 선택        유사도 최고점 1개 선택
 기준값 이상이면 매칭           기준값 이상이면 매칭
        │                              │
        ▼                              ▼
  KC 체인 진입                  CTA 레이어 적용
 (Concept → Risk-type            (KC 체인 결과에
   → Rule → …)                    CTA 추가 노출)
        │                              │
        └──────────────┬───────────────┘
                       ▼
              [최종 답변 + CTA 조합 노출]
```

---

## 결과 처리

### Concept 매칭
- 기준값 이상인 Concept 중 **유사도 최고점 1개만** KC 체인에 진입시킨다.
- 동점이면 카드 ID 오름차순으로 선택한다.
- 기준값 미만이면 Concept 미매칭 → KC 체인 없이 약관 RAG → FAQ RAG → LLM Fallback.

### Playbook 매칭
- 기준값 이상인 Playbook 중 **유사도 최고점 1개만** CTA를 노출한다.
- 기준값 미만이면 CTA 미노출 (KC 답변만 단독 제공).

### 둘 다 매칭될 때
- Concept 매칭 → KC 답변 생성
- Playbook 매칭 → KC 답변 하단에 CTA 추가
- 두 결과가 충돌하지 않으므로 **항상 병렬 노출(additive)**.

### 둘 다 미매칭일 때
- 약관 RAG → FAQ RAG → LLM Fallback 순으로 진행 (answer-logic.md 참조).
- Playbook CTA 없음.

## 미결 항목 (개발팀 결정 필요)

| 항목 | 설명 |
|---|---|
| 기준값 수치 | 유사도 몇 점 이상을 "매칭"으로 볼 건지. Concept·Playbook 각각 별도 설정 가능 여부 포함 |
| 임베딩 모델 | 어떤 모델을 사용할 건지 (한국어 특화 모델 권장) |
| 인덱스 업데이트 타이밍 | 운영자가 카드 저장·라이브 전환 시 즉시 인덱싱 vs. 배치 처리 |
| 동점 처리 | Concept 동점 시 카드 ID 오름차순 외 다른 기준 필요 여부 |
