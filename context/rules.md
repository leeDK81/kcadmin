# KC Admin — 협업 고정 규칙

> 이 파일은 CLAUDE.md에서 분리된 불변 규칙 집합이다. 모든 에이전트가 따른다.

---

## 기본 규칙

- **"시작해"라고 할 때만** 다음 작업(코딩)을 진행한다. 자동 선진행 금지. "시작해" 전까지는 분석·계획·범위 확인만 하고 코드는 건드리지 않는다.
- 단, 작업 실행 여부를 다시 묻지 않는다 ("계속 진행할까요?" 금지) — 사용자가 "시작해"라고 하면 그 즉시 착수한다. 확인은 "시작해" 신호 한 번으로 충분하다.
- `Data/` 폴더는 별도 지시 없이 접근하지 않는다.
- HTML 레이아웃 이상 시: `<div>` / `</div>` 수를 카운트해 불균형 위치를 추적한다.
- 커뮤니케이션 언어: **한국어**. 기술 용어(RAG, LLM 등)는 영문 병기 허용.
- 사용자가 질문하면 애매하게 답하지 말고 **전문가로서 의견(권장안)을 명확히 제시**한다. 선택지만 나열하고 판단을 미루지 않는다.
- **수정 착수 전, 수정 범위를 먼저 말한다.** 요청받은 파일만 보지 말고 연관 파일(단일 진실원 문서, 카드 체인으로 연결된 다른 카드 타입, impact-map.md 대상)까지 찾아본 뒤 "어디부터 어디까지 고칠 것인지"를 먼저 보고하고 착수한다.
- **검수 없이 "완료됐다"고 보고하지 않는다.** 코드 수정 후 아래 관점에서 스스로 검수를 마친 뒤에만 완료를 선언한다:
  - 서비스 정책 정합성 (context/ 진실원 파일과 충돌 없는지)
  - 컨텐츠 정합성 (카드 체인 ID·수치·문구가 실제로 연결되는지)
  - UI 통일성 (design-system.md·ux-patterns.md 패턴 준수)
  - 기능 정합성 (수정한 로직이 관련 화면·카드 타입에 일관되게 반영됐는지)
  - 코딩 버그 (div 불균형, 끊긴 참조, 존재하지 않는 ID 참조 등)

## 단일 진실원 원칙

**"정보는 딱 한 파일에만 존재한다."**

| 정보 종류 | 단일 위치 | 다른 파일에서는 |
|---|---|---|
| 정책 내용 | `context/` 해당 파일 | 파일명만 참조 |
| 금지어·대체어 | `guides/copywriting.md` | 참조 한 줄만 |
| HTML 패턴·CSS | `guides/ux-patterns.md` + `guides/design-system.md` | 파일명만 참조 |
| 에이전트 역할 | `agents/` 각 파일 | — |

agents/ 파일에 정책 내용을 복사하면 즉시 제거한다. 허용되는 것은 "이 정책을 확인하는 체크리스트 항목"뿐.

---

## agents/ 파일 수정 허용 조건 (딱 3가지)

1. 에이전트의 **역할·판단 기준 자체**가 바뀔 때
2. **새 에이전트**를 추가할 때
3. **05(코드검수)·06(기획검수)·08(UI검수)의 체크리스트**에 새 항목을 추가할 때

위 3가지 외에는 agents/ 파일을 수정하지 않는다. 정책이 바뀌면 context/ 파일만 수정한다.

---

## 세션 끝 업데이트 절차

1. 변경된 정책 → 해당 `context/` 파일 수정
2. `context/decisions.md` 변경 이력에 한 줄 추가 (날짜 포함)
3. `context/impact-map.md`에서 해당 정책 행의 영향 파일 목록 확인 → 파생 파일 수정
4. 확정 결정이 HTML에 반영됐으면 git commit + push

레거시 방치 금지. 기존 내용과 충돌 시 구 내용을 교체한다.

---

## 설계 결정 확정 시

- PO(02)가 해당 `context/` 파일을 **즉시** 업데이트한다.
- 미확정 항목은 `context/decisions.md > ## 미결 항목`에 먼저 등록.
- 확정 후 `context/impact-map.md`를 열어 영향 파일을 함께 수정한다.
