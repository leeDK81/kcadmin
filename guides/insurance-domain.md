## Rule 조건 소스 3종

### MYDATA
ICIS(한국신용정보원) 기반 보험 보유 내역. 담보코드 + 담보금액 조건(LTE 이하 / GTE 이상).

### PROMAGE (프롬에이지)
메디에이지 API 기반 개인 건강 데이터. 위험도 항목 + 등급 조건(EQ/GTE/LTE, 2026-07-06 EQ 전용→3종 확장).

위험도 항목: 50개 flat (암 종합 등급 canr_ca, 폐암 lung_ca, 고지혈증, 당뇨병 등 총 50종 단일 목록)
- 생체나이·의료비예측 구현 금지 (폐기됨)

등급 조건: **고위험 > 위험 > 경고 > 주의 > 양호** (심각도순 5단계, 2026-07-06 3단계→확장, 상/중/하 금지). GTE/LTE로 "위험 이상"(고위험·위험 매칭)·"주의 이하"(주의·양호 매칭) 같은 조건을 만들 수 있다.

### 프로파일 (USER_PROFILE_FIELDS)

> 2026-06-25 확정 5개. 직업위험도·출산예정·거주지는 폐기됨.

| 필드코드 | 레이블 | 타입 | 선택값 |
|---|---|---|---|
| age | 나이(만) | range | BETWEEN (시작세~종료세) |
| gender | 성별 | enum | 무관 / 남 / 여 |
| married | 결혼여부 | enum | 무관 / 기혼 / 미혼 |
| has_child | 자녀유무 | enum | 무관 / 있음 / 없음 |
| drives | 운전여부 | enum | 무관 / 유 / 무 |

age 조건: BETWEEN (시작세~종료세 범위 지정)
enum 조건: EQ(같음). 값이 "무관"이면 조건 통과로 처리됨.

## KCD 질병분류체계

한국표준질병·사인분류. 담보코드가 보장하는 질병 범위 정의에 사용.
예: 암진단(A4200) → C00~C97 (악성신생물 전체)
담보코드(보험 상품 coverage 항목) ≠ KCD 코드(의학적 진단 분류) — KCD 보장 범위 정의는 `12_coverage-code-table.html` 담보코드 참조표에서 관리.
