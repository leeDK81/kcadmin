KC Admin 업무 가이드

Clark AI의 정확한 답변을 위한 KC 카드 운영 가이드

이 문서의 목적

KC Admin은 Clark AI가 사용자 질문에 정확하게 답변하도록 판단 기준(KC 카드)을 등록·관리하는 도구입니다. 카드가 없거나 매칭이 안 되면 일반 생성형 AI처럼 답변합니다. 이 가이드를 읽고 순서대로 따라하면 Clark AI의 KC 기반 답변이 사용 가능해집니다.

AI 답변 생성 로직·예시
사용자 질문 → KC 카드 매칭 → 답변 생성까지 전체 흐름. 4가지 답변 케이스(Case 1~4), 카드 선택 우선순위, 출력 필드 연결 범위 스펙과 실제 시나리오 예시를 한 곳에서 확인할 수 있습니다.

연결 유형별 처리 화면
연결 유형 어디서? 조건 · 특이사항
③ Concept → ① Risk-type ③ Concept 편집기 내 Risk-type 선택 패널 라이브 카드만. 선택사항
④ Rule → ① Risk-type ④ Rule 편집기 기본정보 → 대상 Risk-type 선택 라이브 카드만. 필수 — 이 Rule이 어떤 위험 유형을 판정하는지 지정
④ Rule → ② Evidence ④ Rule 편집기 내 Evidence 연결 패널 라이브 Evidence만 · 1개 이상 필수 (검수 게이트)
⑤ Policy → ④ Rule ⑤ Policy 편집기 내 Rule 선택 패널 라이브 카드만. 준법감시인 포함 승인 필수. 선택사항
⑥ Playbook — 발화 키워드·CTA 설정 ⑥ Playbook 편집기 내 발화 키워드·전환 액션 설정 (KC 메인 체인과 독립) 메인 체인과 병렬 독립 동작. 발화 키워드 감지 → KC 매칭 시 CTA 버튼 추가(Case 2), KC 미매칭 시 Standalone 가이드 기반 답변+CTA(Case 4). 공개범위 내부 전용 고정
연결 선택 원칙: 모든 연결 선택 UI에서 라이브 카드만 선택 가능합니다. 임시저장·승인요청 상태 카드는 선택 불가 표시됩니다. Rule → Risk-type 선택·Rule → Evidence 연결은 검수 요청 전 필수이며, 나머지 연결(Concept → Risk-type, Policy → Rule)은 선택사항입니다.
