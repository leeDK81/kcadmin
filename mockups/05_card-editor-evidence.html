Evidence 카드

Rule의 기준값이 왜 그 숫자인지 보여주는 통계 근거입니다. Rule 카드 저장 시 Evidence 1개 이상 연결이 필수이므로 Rule보다 먼저 만드세요.

유형을 선택하면 해당 입력 항목이 표시됩니다.
보닥통계는 조회 조건(담보코드·지표·모집단)만 지정하면 KC 엔진이 자동 산출합니다.
프롬에이지는 카테고리·항목만 지정합니다.
약관 면책사항 고지는 Rule 카드의 "약관 DB 연동" 플래그로 처리됩니다 — 이 편집기에서 별도 등록 불필요.

다음 단계: Evidence 등록 후 → Concept 카드 또는 Rule 카드를 등록하세요.

새 Evidence 카드 임시저장
담당자가 조회 조건(담보코드·지표·세그먼트)을 선택하면, 상담 시 자동으로 보닥 통계 DB에서 최신 수치를 가져옵니다. 수치를 직접 입력하지 않습니다.
모집단: 전체 보닥 사용자 (세그먼트 구분 없음)
현재 상태: 임시저장 — 담보코드·조회 지표를 선택하고 활용 방법을 입력한 후 [검수 요청]을 클릭하세요.
민감정보 처리 원칙 적용 (§3-8)
  • 건강검진 원본 데이터는 보닥 서버(국내)에만 저장
  • 사용자 출력 시 3단계 등급(위험 / 주의 / 양호)만 표시 — 수치·percentile 노출 금지
  • 정보보호실 사전 협의 필요 항목 확인 후 등록
메디에이지 건강검진 분석 결과 — 해당 고객의 건강 위험 등급을 데이터 출처에서 가져옵니다. 카테고리와 항목만 선택하면 됩니다.
AI 답변 예시: "개인 암 위험도가 높은 편이라는 점을 감안하면, 암 담보 보강을 우선적으로 검토하시길 권합니다."
현재 상태: 임시저장 — 카테고리·항목을 선택한 후 [검수 요청]을 클릭하세요.
위에서 Evidence 유형을 선택하면 입력 폼이 표시됩니다
