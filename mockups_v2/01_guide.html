KC Admin 업무 가이드

Clark AI의 정확한 답변을 위한 KC 카드 운영 가이드

이 문서의 목적

KC Admin은 Clark AI가 사용자 질문에 정확하게 답변하도록 판단 기준(KC 카드)을 등록·관리하는 도구입니다. 카드가 없거나 매칭이 안 되면 RAG 레이어가 먼저 동작합니다. 약관 RAG(가입 상품 약관 자동 파싱)와 FAQ RAG(Clark 서비스 전용 Q&A)가 순서대로 검색되며, RAG 결과도 없을 때만 제한된 Fallback 생성형 AI 답변이 나옵니다. 이 가이드를 읽고 순서대로 따라하면 Clark AI의 KC 기반 답변이 사용 가능해집니다.

답변 우선순위: KC 체인 기반 정밀 답변 (Case 1·2) → RAG 기반 답변 (Case 3 RAG 성공) → Fallback 제한 생성형 (Case 3 RAG 실패) / Playbook Standalone (Case 4)

AI 답변 생성 로직·예시
사용자 질문 → KC 카드 매칭 → 답변 생성까지 전체 흐름. 4가지 답변 케이스(Case 1~4), 카드 선택 우선순위, 출력 필드 연결 범위 스펙과 실제 시나리오 예시를 한 곳에서 확인할 수 있습니다.

Risk-type → Rule 카드 캔버스에서 연결 승인완료 이상. 필수 — 이 위험 유형에 적용할 판단 Rule 지정
Rule → Evidence 카드 캔버스에서 연결 승인완료 이상 Evidence · 1개 이상 필수 (근거 데이터 확보)
Rule → Policy 카드 캔버스에서 연결 승인완료 이상. 준법감시인 포함 승인 필수. 선택사항
Playbook — 발화 키워드·CTA 설정 Playbook 편집기 내 발화 키워드·전환 액션 설정 (KC 메인 체인과 독립) 메인 체인과 병렬 독립 동작. 발화 키워드 감지 → KC 매칭 시 CTA 버튼 추가(Case 2), KC 미매칭 시 Standalone 가이드 기반 답변+CTA(Case 4). 공개범위 내부 전용 고정
연결 원칙: 카드 캔버스에서 승인완료(approved) 이상 카드만 연결 가능합니다. 승인완료 카드 간 연결은 준비 연결(점선 파랑)로 표시되며, 라이브 전환 후 Clark AI에 반영됩니다. 임시저장(draft) 카드만 캔버스에 표시되지 않습니다. 검수요청(review) 카드는 캔버스에 표시되나 연결이 불가합니다. 카드 편집기에서는 연결 상태를 읽기 전용으로 확인할 수 있으며, 수정은 캔버스에서만 가능합니다.
카드 유형 — 편집기 바로가기

각 카드는 순서 없이 독립적으로 등록·승인할 수 있습니다. 승인완료된 카드부터 카드 캔버스에서 Concept → Risk → Rule → Evidence·Policy 순서로 연결하세요.

Risk-type
사용자 보장 공백 유형을 정의합니다. AI 답변 체인의 중심 분류 기준.
체인 위치: Concept → Risk → Rule
목록으로 →
Evidence
공인 외부 통계를 직접 입력합니다 (HIRA·NHIS·KOSTAT·FSS). Rule의 근거 데이터.
체인 위치: Rule → Evidence
목록으로 →
Concept
AI 답변 체인의 진입점. 사용자 질의 키워드와 동의어 매칭.
체인 위치: Concept → Risk
목록으로 →
Rule
고객 보장 현황 데이터 기반으로 보장 공백 여부를 판단하는 조건 설정.
체인 위치: Risk → Rule → Evidence·Policy
목록으로 →
Policy
준법 고지 문구·출력 제한 설정. 2단계 승인 필요 (도메인 + 준법감시인).
체인 위치: Rule → Policy  ·  선택사항
목록으로 →
Playbook
