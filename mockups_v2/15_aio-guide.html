AIO 가이드

외부 AI 시스템이 Clark KC 카드 콘텐츠를 참조하는 방식 — 개발자·운영자 공통 문서

AIO(AI Indexing Optimization)는 SEO가 검색엔진에 콘텐츠를 노출하는 것처럼, AI 시스템(Google AI Overviews, Perplexity, ChatGPT 등)이 Clark의 KC 카드 콘텐츠를 참조·인용하도록 최적화하는 전략입니다.
KC 카드 중 공개범위가 "고객 공개"인 카드만 AIO 색인 대상입니다(Risk-type·Concept). Evidence·Policy·Playbook은 공개범위가 "내부 전용"이므로 AIO 색인에서 자동 제외됩니다. Rule은 공통 기준(badge-baseline)으로 고객 출력이 없어 AIO 색인에서 제외됩니다.
AIO란? — 운영자를 위한 설명

사용자가 "암보험 얼마나 있어야 해?"를 Google이나 ChatGPT에 물었을 때, Clark의 KC 카드에 등록된 정보가 AI 답변에 인용될 수 있습니다.
이를 위해서는 KC 카드 콘텐츠가 공개 URL로 게시되고, AI가 이해할 수 있는 구조로 마크업되어 있어야 합니다.

AI가 콘텐츠를 참조하는 3가지 방식
메커니즘 1
웹 크롤링 → 학습 데이터
ChatGPT, Claude 등 LLM이 학습할 때 공개 인터넷을 크롤링합니다. 공개 URL로 접근 가능한 KC 카드 콘텐츠는 다음 모델 버전 학습에 포함될 수 있습니다.

허용 설정: robots.txt에서 GPTBot, ClaudeBot, Google-Extended 크롤러를 허용해야 합니다.
⏱ 시차 있음 — 다음 학습 때 반영
메커니즘 2
실시간 RAG
Google AI Overviews, Perplexity, ChatGPT 검색 모드는 질문이 들어오면 실시간으로 웹 검색 후 콘텐츠를 참조해 답변을 생성합니다.

핵심: Schema.org 구조화 데이터(FAQ, HowTo)가 있으면 AI가 콘텐츠 의미를 더 정확히 파악해 인용 가능성이 높아집니다.
⚡ 즉시 반영 — 가장 중요
메커니즘 3
llms.txt
robots.txt의 AI 버전입니다. 사이트 루트에 /llms.txt 파일을 두면 LLM 크롤러에게 "이 페이지들을 참조하세요"라고 안내할 수 있습니다.

빠르게 도입 가능: 파일 하나로 모든 주요 AI 크롤러에게 Clark의 핵심 보험 정보 페이지를 안내합니다.

KC 카드 → AIO 활용 필드 연결

AIO 색인 대상인 고객 공개 카드(Concept·Risk-type)의 필드별 활용 방식을 정의합니다.

카드 유형 필드 AIO 활용 노출 여부
Concept term (용어명) FAQ 질문 키워드. "암보험이란?", "실손보험이란?" 형태로 질문 구성 노출
definition (한 줄 정의) FAQ 답변 리드 문장. 간결한 정의로 AI 인용 가능성 높음 노출
Risk-type name (유형명) 답변 제목. "암투병생활비 부족형이란?" 형태의 질문 생성 노출
description (설명) FAQ 답변 본문. 위험 유형의 정의와 의미 설명 노출
주의 — Evidence 공개범위: Evidence 카드(공인 외부 통계)는 공개범위가 내부 전용으로 고정됩니다 — 운영자 변경 불가. 이에 따라 출처 기관명·지표명·기준값은 AIO 색인 대상이 아닙니다. 프롬에이지 개인 건강 데이터는 Evidence 카드와 무관하며, 상담 시 자동 전달되는 별도 채널로 처리됩니다 [§3-8].
Schema.org 구현 가이드 — 개발자용

KC 카드 퍼블리시 파이프라인에서 아래 JSON-LD를 자동 생성해 각 FAQ 페이지 <head>에 삽입합니다.

FAQPage 스키마 — Concept + Risk-type 카드 조합
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "FAQPage", "mainEntity": [ { "@type": "Question", "name": "암보험 진단금은 얼마가 적당한가요?", ← Concept.term + synonyms 기반 자동 생성 "acceptedAnswer": { "@type": "Answer", "text": "암투병생활비 부족형은 암 진단 후 치료·생활비 부담이 높아지는 위험 유형입니다. 40대 기준 진단금 5,000만원 이상을 권장합니다." ← Risk-type.description + Evidence.usageNote 조합 } } ] } </script>
robots.txt — AI 크롤러 허용 설정
# 허용: Google AI (AI Overviews) User-agent: Google-Extended Allow: / # 허용: OpenAI User-agent: GPTBot Allow: / # 허용: Anthropic Claude User-agent: ClaudeBot Allow: / # 허용: Perplexity User-agent: PerplexityBot Allow: / # 차단: 학습 데이터 수집 전용 크롤러 (선택적) # User-agent: CCBot # Disallow: /
llms.txt 설정 가이드 — 개발자용

사이트 루트(clark.ai/llms.txt)에 배치합니다. KC 카드 "고객 공개" 콘텐츠 페이지 목록(Risk-type·Concept)을 AI 크롤러에게 직접 안내합니다.

KC 카드가 라이브 전환되고 공개범위가 "고객 공개"이면(Risk-type·Concept), 퍼블리시 파이프라인이 llms.txt에 해당 카드 항목을 자동 추가합니다. 운영자가 직접 편집할 필요 없습니다.
