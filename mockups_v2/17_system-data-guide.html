Clark AI가 자동으로 활용하는 데이터

마이데이터·프롬에이지·프로파일은 상담 시 자동으로 Clark AI에 전달됩니다 — Evidence 카드로 별도 등록하지 않아도 됩니다. 공인 외부 통계(심평원·건보공단 등)는 Clark AI가 자동 접근할 수 없으므로 운영자가 Evidence 카드로 직접 등록해야 합니다. 약관 DB는 크롤링 파이프라인이 자동 관리하며, 약관 DB 확보 시 자동으로 조회됩니다.

마이데이터 + 프롬에이지 + 프로파일 자동 전달
  • 마이데이터 — coverage_amt (보장 금액) / renewal_status (갱신 여부) / insu_type (보험 유형) / start_date (가입일) / end_date (만기일)
  • 프롬에이지 — 건강 위험도 등급 (고위험/위험/경고/주의/양호)
  • 프로파일 — age (나이: 시작세~종료세 범위) / gender (성별: 무관/남/여) / married (결혼여부: 무관/기혼/미혼) / has_child (자녀유무: 무관/있음/없음) / drives (운전여부: 무관/유/무)
상담 시 실시간 전달. Rule 조건 판단에 직접 활용. Evidence 카드 등록 불필요.
공인 외부 통계 Evidence 카드 등록 필요
  • HIRA(심평원) — 암·뇌졸중·심혈관 평균 입원 진료비
  • NHIS(건보공단) — 발병률·5년 생존율·의료이용 통계
  • KOSTAT(통계청) — 사망원인·기대여명
  • FSS(금감원) — 보험계약 유지율·미가입률 통계
Clark AI가 자동 접근 불가 — 운영자가 출처·지표·기준값을 Evidence 카드에 직접 입력해야 합니다.
약관 DB 파이프라인 자동 관리
  • 보험사별 담보 약관 원문 (크롤링 파이프라인 자동 수집)
  • 담보별 주요 면책 항목 (면책기간·기왕증·특정 치료 제한 등)
  • 약관 개정 이력 및 적용 기간
보험사별·가입 시점별로 약관이 다를 수 있습니다. 약관 DB 확보 시 자동으로 담보코드 기준으로 조회됩니다.
카드 유형별 데이터 연결
카드 유형 참조하는 데이터 역할 데이터 시점
② Risk-type 사용자 마이데이터 (보험 보유 현황) 사용자의 보장 공백 유형을 분류 (T01~T23) 상담 시 실시간
④ Evidence — 공인 외부 통계 HIRA·NHIS·KOSTAT·FSS 등 공인 기관 통계 (운영자 직접 입력) "암 평균 입원 진료비 3,800만원(심평원 2024)" 등 Rule 판단의 금액·비율 기준 근거 제공. 금소법상 설명 의무 이행 근거로 활용 운영자가 Evidence 카드 등록 시 고정값 (연 1회 이상 갱신 권장)
③ Rule — 약관 DB 자동 참조 약관 DB (크롤링 파이프라인 자동 관리) 약관 DB 확보 시 Rule 조건 충족 → 담보코드 기준으로 자동 조회 → Clark 답변에 ⚠️ 면책조항 섹션 자동 포함. 마이데이터 금액만으로 "다 보장" 오해 방지 파이프라인이 약관 개정 시 자동 최신화
① Concept 사용자 질의 텍스트 질의와 관련된 KC 카드를 연결. 없으면 생성형 AI로 답변 질의 시 실시간
③ Rule 마이데이터 + 프롬에이지 + 프로파일 "암진단 담보가 3,000만 원 미만인 경우" 같은 조건 판단 상담 시 실시간
⑤ Policy 없음 출력 규칙 적용 (특정 표현 금지·면책 문구 추가 등) 규칙 내용만 적용
⑥ Playbook 사용자 발화 키워드 (KC 메인 체인과 독립 병렬 감지) 발화 키워드 감지 시 리드 전환 CTA 버튼 출력. Case 4(KC 미매칭)에서는 Standalone 답변 가이드 기반 AI 답변 생성 후 CTA 제공 발화 감지 즉시 (병렬)
Clark AI에 전달되는 정보와 전달되지 않는 정보
전달되는 정보

Clark AI는 아래 정보를 상담 시 전달받아 분석에 활용합니다.

  • ✓ 보험 담보 종류 (예: 암진단, 뇌졸중, 입원일당)
  • ✓ 담보별 보장 금액 범위
  • ✓ 납입 방식 (갱신형/비갱신형)
  • ✓ 프롬에이지 건강 위험도 등급 (고위험/위험/경고/주의/양호)
  • ✓ 나이, 성별, 결혼여부, 자녀유무, 운전여부 기본 프로파일 5개 항목
전달되지 않는 정보

Clark AI는 아래 정보를 전달받지 않으며, 분석에도 활용하지 않습니다.

  • ✕ 이름, 주민등록번호, 연락처
  • ✕ 보험사 이름, 보험 상품명, 계약 번호
  • ✕ 프롬에이지 원본 건강 수치 (수치 대신 등급만 전달)
  • ✕ 보험금 청구 이력 (설계 의도적 미활용)
프롬에이지 건강 정보는 민감 개인정보이므로, Clark 답변에서도 원본 수치는 절대 출력하지 않습니다. 고위험/위험/경고/주의/양호 등급명과 백분위 형태로만 제공됩니다(2026-07-06 3단계→5단계 확장). [§3-8 준수]
이것만은 꼭 확인해 주세요
유의 ① Evidence에는 공인 기관 통계를 직접 입력해야 합니다
Evidence 카드는 Clark AI가 자동으로 접근할 수 없는 공인 외부 통계 전용입니다. 운영자가 출처 기관·보고서명·기준 연도·지표명·기준값을 직접 입력해야 합니다. 마이데이터·프롬에이지는 상담 시 자동 전달되므로 별도 Evidence 카드 등록이 필요하지 않습니다.
유의 ② Evidence 통계는 기준 연도를 반드시 명시하고 주기적으로 갱신해야 합니다
공인 기관 통계는 연도별로 수치가 바뀝니다. Evidence 카드 등록 시 기준 연도를 명확히 입력하고, 신규 발표가 있을 때마다 카드를 업데이트해야 합니다. 1년 이상 경과한 통계를 근거로 사용하면 Clark 답변의 신뢰도가 하락합니다.
유의 ③ Rule 조건의 프롬에이지 — 등급 기준만 가능, 선택 조건으로 설정 권장
Rule 조건에서 프롬에이지 항목을 사용할 때는 수치가 아닌 등급(고위험>위험>경고>주의>양호 5단계, 2026-07-06 3단계→확장)으로만 조건을 설정할 수 있습니다. "위험도가 위험 등급인 경우"(EQ)뿐 아니라 "위험도가 위험 등급 이상인 경우"(GTE, 고위험·위험 매칭)·"주의 등급 이하인 경우"(LTE, 주의·양호 매칭)도 가능합니다. 다만 "위험도 점수가 85점 이상인 경우"처럼 원본 수치로 조건을 설정하는 것은 여전히 불가합니다.

프롬에이지 조건은 선택 조건으로 설정하세요. 필수 조건으로 설정하면 프롬에이지 미연동 사용자는 해당 Rule이 발동하지 않습니다. 선택 조건으로 설정하면 미연동 사용자도 Rule이 발동하며, 프롬에이지 충족 여부는 Risk-type 노출 우선순위(2순위: 선택 조건 충족 개수)에만 반영됩니다.
유의 ④ Evidence 없는 Rule은 근거 없는 답변을 만듭니다
Rule에 Evidence가 연결되지 않아도 조건 판단 자체는 작동합니다. 단, 답변에서 "왜 이 보험이 부족한지"를 설명하는 공인 통계 근거 섹션이 빠지므로 Clark이 권고만 하게 됩니다. Rule을 등록할 때 관련 공인 외부 통계 Evidence를 함께 연결해 주세요.
유의 ⑤ 약관 DB 파이프라인은 파이프라인 팀이 관리합니다
약관 DB는 크롤링 파이프라인이 자동으로 수집·최신화하며, 약관 DB가 확보된 담보코드에 대해 Clark 답변에 면책조항 섹션이 자동 포함됩니다. 파이프라인이 최신 약관을 정확히 수집·파싱하고 있는지 파이프라인 담당자와 주기적으로 확인하세요. 약관은 보험사별·가입 시점별로 다를 수 있으므로 Clark 답변에는 "실제 지급 여부는 보험사 확인 필요" 안내 문구가 반드시 함께 출력되도록 Policy 카드에서 설정해 주세요.
자주 묻는 질문
Q1. 보닥에 마이데이터를 연동하지 않은 사용자의 경우 어떻게 되나요?
마이데이터 미연동 사용자는 보험 보유 현황 데이터가 없으므로, 마이데이터 기반의 Rule 조건이 충족되지 않습니다. 이 경우 프로파일(나이·성별 등)만으로 판단하는 Rule이나 Playbook(Case 4)으로 답변이 구성됩니다.
Q2. 프롬에이지를 측정하지 않은 사용자는요?
프롬에이지 조건이 필수 조건으로 설정된 Rule은 미연동 사용자에게 발동하지 않습니다. 반면 선택 조건으로 설정된 경우 Rule은 정상 발동하며, 해당 선택 조건은 충족되지 않은 것으로 처리됩니다(Risk-type 우선순위 2순위인 선택 조건 충족 개수에 미포함). 이 때문에 프롬에이지 조건은 선택 조건으로 설정하는 것을 권장합니다. 마이데이터·프로파일 기반의 다른 카드들은 정상 작동하며, 공인 외부 통계 기반 Evidence는 프롬에이지 유무와 관계없이 항상 참조됩니다.
Q3. Evidence 카드에 입력하는 공인 통계는 얼마나 자주 갱신해야 하나요?
심평원·건보공단·통계청·금감원의 주요 통계는 보통 연 1회 발표됩니다. 신규 통계가 발표되면 해당 Evidence 카드의 기준 연도·기준값을 업데이트하고 검수 요청을 통해 재승인을 받아야 합니다. 1년 이상 경과한 통계를 계속 사용하면 Clark 답변의 신뢰도가 하락할 수 있습니다. Evidence 카드 목록에서 기준 연도를 확인하고 오래된 카드를 주기적으로 점검해 주세요.
Q4. 약관 DB 연동으로 나오는 면책조항이 실제 보험사 약관과 완전히 동일한가요?
완전히 동일하지 않을 수 있습니다. 약관 DB는 크롤링 파이프라인이 보험사 공시 약관을 수집·파싱한 데이터로, 세부 조건은 가입 시점·보험사별로 다를 수 있습니다. Clark은 이를 "보장 제한 가능성 고지" 용도로만 활용하며, 실제 지급 여부의 최종 판단은 보험사에 있습니다. 이 안내 문구는 Policy 카드에서 설정해 Clark 답변에 반드시 포함시켜야 합니다.
