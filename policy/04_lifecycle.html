상태별 상세 정의

임시저장 (draft)

운영자가 카드를 작성 중인 상태. 언제든 저장·수정 가능. 검수 요청 전까지 이 상태를 유지한다. 캔버스에 표시되지 않으며 연결 불가.

전환 조건: 운영자가 [검수 요청] 버튼 클릭 → review 전환 / 반려 후 [재편집] → draft 복원

승인요청 (review)

운영자가 검수를 요청한 상태. 검수자가 내용을 검토한다. 이 상태에서 운영자는 수정 불가(요청 취소 후 draft로 돌아가야 수정 가능). 캔버스에 표시되나 연결 불가.

전환 조건: 검수자 승인 → approved / 검수자 반려 → rejected / 운영자 요청 취소 → draft

승인완료 (approved)

검수자가 내용을 승인한 상태. 캔버스에서 연결 가능(준비 연결, pending 엣지). Clark AI에는 아직 미반영. 사전 테스트(Dry-run) 후 배포 요청으로 라이브 전환 가능.

Policy 카드 특이사항: 도메인 검수자 승인 후 준법감시인 추가 승인 필요. 두 단계 모두 완료해야 approved 상태.

Playbook 카드 특이사항: Playbook은 다른 카드와 캔버스 연결을 하지 않는다(CONNECT_RULES 미적용, 독립 체인). 따라서 approved 상태에서 캔버스 연결 없이 바로 "라이브 전환" 버튼이 노출된다. 사전 테스트 단계도 생략 가능.

전환 조건 (일반 카드): 캔버스 연결 → 사전 테스트 통과 → 배포 요청 → active
전환 조건 (Playbook): 승인완료 → "라이브 전환" 버튼 클릭 → active (캔버스 연결·사전 테스트 불필요)

라이브 (active)

배포 완료 상태. Clark AI에 즉시 반영. 연결된 모든 카드가 active가 되면 엣지도 active(실선 초록)로 전환. 운영 중 수정이 필요하면 [편집] → 내용 수정 → 재검수 → 재배포 흐름.

전환 조건: [일시중지] → paused

일시중지 (paused)

운영자 또는 관리자가 카드를 일시적으로 비활성화한 상태. Clark AI에서 즉시 제외. 기존 연결은 유지되나 AI 답변에 활용되지 않는다. [재시작] → active로 즉시 복원 가능.

전환 조건: [재시작] → active

반려 (rejected)

검수자가 내용을 반려한 상태. 운영자가 반려 코멘트를 확인하고 내용을 수정한 뒤 다시 검수 요청해야 한다. 캔버스에 표시되지 않음.

전환 조건: 운영자 [재편집] → draft 전환 후 수정 → review 재요청

Policy 카드 2단계 반려 특이사항: 도메인 검수자 또는 준법감시인 중 한 단계에서 반려되어도 전체 반려. 단, 이미 통과한 단계는 재검토 불필요. 준법감시인이 반려한 경우 수정 후 도메인 검수자 재검토 없이 준법감시인 단계부터 재개.
