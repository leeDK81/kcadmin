05 승인 정책

1단계 표준 검수·승인 흐름 및 Policy 카드 2단계 승인 정책

표준 1단계 검수·승인 흐름 (Risk-type / Evidence / Concept / Rule / Playbook)
①

내용 입력 및 임시저장

운영자가 카드 편집기에서 필수 필드를 입력하고 임시저장한다. 임시저장 상태에서는 언제든 수정 가능.

담당: 운영자 / 상태: draft
②

검수 요청

운영자가 [검수 요청] 버튼을 클릭. 카드 상태가 draft → review로 전환. 검수자에게 알림 발송. 이후 운영자는 직접 수정 불가(취소만 가능).

담당: 운영자 / 상태: review
③

검수자 검토

검수자가 카드 내용을 검토한다. 검수·승인 화면(09_review-workflow.html)에서 처리. 코멘트 입력 후 승인 또는 반려.

담당: 검수자
④

승인완료

검수자 승인 시 카드 상태가 review → approved로 전환. 카드 캔버스에서 연결 구성 가능(준비 연결). Clark AI에는 미반영.

담당: 검수자 / 상태: approved
⑤

캔버스 연결 → 사전 테스트 → 라이브 전환

카드 캔버스에서 연결 구성 후 사전 테스트 통과 시 [배포 요청]으로 라이브 전환. 상태: approved → active. Clark AI 즉시 반영.

담당: 운영자 / 상태: active
Policy 카드 2단계 승인 흐름
Policy 카드는 준법 고지 문구를 포함하므로 도메인 검수자 승인 후 준법감시인의 추가 최종 승인이 필요합니다. 두 단계 모두 완료해야 approved 상태로 전환됩니다.
①②

내용 입력 → 검수 요청

표준 흐름과 동일. 운영자가 Policy 내용(Policy 이름·Clark 앱 표시 문구) 입력 후 검수 요청.

담당: 운영자
③

도메인 검수자 검토

도메인 전문 검수자가 Policy 내용의 정확성·적절성을 검토. 1차 승인 처리.

담당: 도메인 검수자
④

준법감시인 최종 승인

준법감시인이 준법 고지 문구의 적법성·금소법 준수 여부를 검토하여 최종 승인. 이 단계 없이 approved 전환 불가.

담당: 준법감시인
⑤

승인완료 → 캔버스 연결 → 라이브

2단계 승인 완료 시 approved 상태. 이후 캔버스 연결·사전 테스트·라이브 전환 흐름은 표준과 동일.

상태: approved → (연결·테스트 후) active

승인완료 후 → 라이브 전환 흐름

일반 카드 (Concept / Risk-type / Rule / Evidence / Policy): 승인완료(approved) 카드는 아직 Clark AI에 반영되지 않습니다. 라이브 전환 전 아래 단계를 반드시 거쳐야 합니다.

  1. 캔버스 연결 구성: 카드 캔버스에서 CONNECT_RULES에 따라 카드 간 연결 설정 (준비 연결·점선 파랑)
  2. 사전 테스트: 실제 고객 프로필로 Rule 조건 테스트. 예상 AI 답변 경로 확인
  3. 배포 요청: 사전 테스트 통과 후 [배포 요청] 클릭 → active 전환 → Clark AI 즉시 반영

Playbook 카드 (예외): Playbook은 CONNECT_RULES 미적용 — 독립 체인으로 동작하므로 캔버스 연결 없이 라이브 전환이 가능합니다.

  1. 승인완료 확인: 검수·승인 화면(09_review-workflow.html)에서 1단계 검수자 승인
  2. 라이브 전환: approved 상태에서 "라이브 전환" 버튼 클릭 → active 전환 (캔버스 연결·사전 테스트 불필요)

Playbook 라이브 전환 주체: 운영자 또는 관리자.

일반 카드는 승인완료 후 캔버스 연결 없이 라이브 전환하는 경로가 없습니다. 연결 없는 단독 카드는 Clark AI 답변 체인에서 활용되지 않습니다. Playbook은 이 규칙의 예외입니다.
