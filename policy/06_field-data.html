06 필드·데이터 정의

6종 KC카드 타입별 필드 목록, 필수/선택, 데이터 타입 및 유효성 규칙

Risk-type 카드 필드
필드명API 키필수타입설명 / 유효성
카드 코드code필수stringT01~T10 형식. 시스템 자동 부여. 편집 불가
명칭name필수string최대 30자. 예: "암투병생활비 부족형"
설명desc필수string최대 200자. 이 위험 유형이 발생하는 상황 설명
중요도priority필수enumhigh / mid / low. 기본값: mid(보통). 다중 감지 시 노출 우선순위 1순위 기준: 높음 > 보통 > 낮음 (동점 시 → 2순위: 선택 조건 충족 개수 / 3순위: 카드 코드 오름차순)
리드 스코어leadScore선택numberPhase 1.5 이후 활성화. 현재 미사용
공개범위scope자동string"public" 고정. 운영자 변경 불가
상태status자동enumdraft / review / approved / active / paused / rejected

Concept 카드 필드
필드명API 키필수타입설명 / 유효성

연결 Risk-typelinkedRiskTypes필수string[]캔버스에서 설정. 편집기에서 읽기 전용 표시. 미연결 시 KC 체인 미진입
공개범위scope자동string"public" 고정
Rule 카드 필드
필드명API 키필수타입설명 / 유효성
카드 코드code필수stringRU001~ 형식. 시스템 자동 부여
Rule명name필수string최대 60자
설명desc필수string이 Rule이 판단하는 내용
판단 조건conditions필수ConditionRow[]최소 1개. 3개 데이터 소스(MYDATA·PROMAGE·PROFILE) 중 선택하여 조건 행 배열 구성. 각 행에 필수/선택 구분(required: boolean) 설정 필요. PROMAGE 조건은 선택 권장
출력 등급outputGrade필수enum위험 / 주의 / 양호 (상/중/하 사용 금지)
연결 Risk-typelinkedRiskTypes캔버스string[]캔버스에서 설정. 편집기 읽기 전용
연결 EvidencelinkedEvidences캔버스string[]캔버스에서 설정. 최소 1개 필요
연결 PolicylinkedPolicies캔버스string[]캔버스에서 설정. 선택사항
공개범위scope자동string"baseline" 고정
ConditionRow 구조: { source: "MYDATA"|"PROMAGE"|"PROFILE", field: string, operator: string, value: string, required: boolean }.
· required: true — 필수 조건. 미충족 시 Rule 미발동 (AND)
· required: false — 선택 조건. 미충족 시에도 Rule 발동. 충족 개수는 Risk-type 노출 우선순위 2순위에 활용
· MYDATA는 추가로 coverageCode 필드 포함. PROMAGE 조건은 required: false 권장 (미연동 사용자 대응)
MYDATA 조건 항목: coverage_amt(담보금액) 1종만 허용. 갱신여부·보험유형·시작일·종료일 조건 구현 금지.

Rule 조건 빌더 — 데이터 소스 3종 구조
소스source 값행 구조필수/선택 기본값배경색 클래스
마이데이터 MYDATA 담보코드 select + 담보금액(coverage_amt, 정적 레이블) + 조건 select(LTE/GTE/LT/GT/EQ) + 기준값 + 필수/선택 라디오 required: true (필수) .cond-row-mydata (기본 회색)
프롬에이지 PROMAGE 위험도 항목 select(50개, optgroup: 암위험도 23개 / 질병위험도 27개) + 같음(EQ 고정) + 등급 select(위험/주의/양호) + 필수/선택 라디오. 생체나이·의료비예측 구현 금지. required: false (선택 — 미연동 사용자 대응) .cond-row-promage (보라, #FAF8FD)
프로파일 PROFILE 항목 select(4개: age·gender·married·has_child) + 조건 + 기준값 + 필수/선택 라디오 required: true (필수) .cond-row-profile (초록, #F4FBF8)
Amplitude 소스는 완전 제거됨(2026-06-08). 조건 빌더에 4번째 소스 구현 금지.
