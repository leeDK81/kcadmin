08 화면 정책

KC Admin 전체 24개 화면의 목적, 구성, 구현 시 준수 사항

모든 화면의 목업은 mockups_v2/ 폴더에 위치. 사이드바 구조·CSS 변수·컴포넌트 패턴은 guides/design-system.md, guides/ux-patterns.md 참조. 카드 연결 UI(피커·드롭다운)는 편집기에 구현하지 않음 — 캔버스 전용.
파일명 화면명 목적 및 주요 구성 구현 시 준수 사항
공통 · 시스템
00_canvas-main.html 카드 캔버스 승인완료 이상 카드를 시각적으로 연결하는 핵심 화면. 상단 피커(4컬럼 — CONCEPT·RISK-TYPE·RULE·EVIDENCE·POLICY, Playbook 제외)에서 카드 선택 → 하단 그리드에서 연결 추가·제거. Pending/Active 엣지 구분 표시. 테스트 모드(툴바 전환) — 개별 카드 체크 후 "테스트 실행"으로 Rule 조건 매칭·예상 AI 답변 경로를 미리 확인(舊 11_dry-run.html 기능이 이 화면으로 통합됨, 2026-06-25). 카드 연결은 이 화면에서만 가능. 편집기에 연결 UI 구현 금지. draft 카드 미표시. 사전 테스트는 테스트 모드에서만 진행(연결 모드 슬라이드 패널엔 탭 없음, 2026-07-05). 화면명 "사전 테스트" 사용, "시뮬레이션" 금지.
00_design-system.html 디자인 시스템 CSS 변수·색상·타이포그래피·컴포넌트 레퍼런스. 개발자·디자이너 공용 참조 화면. 실제 서비스 화면 아님. 목업 내 참조용.
01_guide.html 업무 가이드 KC Admin 전체 운영 프로세스 안내. 5단계 흐름·카드 유형별 바로가기·연결 유형 테이블. 사이드바 "업무 가이드" 항목 링크 대상.
02_dashboard.html 대시보드 KC Admin 전체 현황 요약. 카드 유형별 등록 수·상태별 분포·라이브 커버리지 지표. "시뮬레이션" 표현 금지 → "사전 테스트"로 표기.
03_card-library.html 카드 라이브러리 6종 카드 유형을 탭으로 전환하며 전체 목록 조회. 유형별 신규 등록 진입점. 공개범위 배지(고객 공개/공통 기준/내부 전용)는 목록 화면에서만 표시.
Risk-type
04_risk-type-list.html Risk-type 목록 전체 Risk-type 카드 목록. 상태·중요도 배지, 코드·명칭, 연결 Rule 요약 표시. 중요도 컬럼(높음/보통/낮음 배지) 포함. table-layout:fixed 적용.
04_card-editor-risk-type.html Risk-type 편집기 Risk-type 카드 신규 등록·수정. 기본정보(명칭·설명·중요도 라디오) + 액션 카드(임시저장·검수 요청) + wf-tracker. 중요도 라디오: 높음/보통(기본)/낮음 (가중치 없음, 서열만). 연결 Rule은 읽기 전용 배지 + "캔버스에서 변경" 버튼.
Evidence
05_evidence-list.html Evidence 목록 전체 Evidence 카드 목록. 출처 기관 배지, 기준 연도, 연결 Rule 요약 표시. 출처 유형 select 없음(단일 유형). table-layout:fixed 적용.
05_card-editor-evidence.html Evidence 편집기 Evidence 카드 신규 등록·수정. 출처 기관·보고서명·기준 연도·지표명·기준값·활용 방법 필드. 출처 기관 select: HIRA/NHIS/KOSTAT/FSS/기타. 유형 select 미표시(단일 유형).
Concept
06_concept-list.html Concept 목록 전체 Concept 카드 목록. 연결 Risk-type 수, 동의어 수 표시. table-layout:fixed 적용.
06_card-editor-concept.html Concept 편집기 Concept 카드 신규 등록·수정. 개념명·설명·동의어 목록 + 연결 Risk-type(읽기 전용) + 액션. 연결 Risk-type: 읽기 전용 배지 + "캔버스에서 변경" 버튼. Standalone 라디오 없음(완전 제거).
Rule
07_rule-list.html Rule 목록 전체 Rule 카드 목록. 연결 Risk-type·Evidence·Policy 요약 표시. table-layout:fixed 적용.
07_card-editor-rule.html Rule 편집기 Rule 카드 신규 등록·수정. 기본정보 + 조건 빌더(MYDATA·PROMAGE·PROFILE, 필수/선택 구분) + Evidence 연결(읽기 전용) + 액션. 조건 빌더 데이터 소스: MYDATA·PROMAGE·프로파일(Amplitude 없음). 등급명: 고위험/위험/경고/주의/양호(2026-07-06 3단계→5단계 확장, 상/중/하 금지).
Policy
08_policy-list.html Policy 목록 전체 Policy 카드 목록. 이름·Clark 앱 표시 문구·연결 Rule 표시. 2단계 승인 상태 표시 필요. table-layout:fixed 적용.
08_card-editor-policy.html Policy 편집기 Policy 카드 신규 등록·수정. Policy 이름 + Clark 앱 표시 문구 (2개만). 연결 Rule: 읽기 전용 배지 + "캔버스에서 변경". 준법감시인 2단계 승인 상태 표시. Clark 앱 표시 문구 비어있으면 검수 요청 불가.
검수·배포·운영
09_review-workflow.html 검수·승인 검수자·준법감시인이 승인 요청된 카드를 검토하는 화면. 카드 상세 확인 + 코멘트 + 승인/반려 액션. 승인완료 상태의 액션: "캔버스에서 연결" 버튼(라이브 전환 버튼 없음). 승인요청 카드에 연결 정보(conn) 표시 금지. 승인완료 카드는 conn 표시 허용 — 이미 캔버스에서 연결된 카드일 수 있으므로. Playbook은 예외: approved 상태에서 캔버스 연결 없이 바로 "라이브 전환" 버튼 노출.
18_system-settings.html 시스템 설정 LLM Fallback 정책(KC 미매칭 + RAG 결과 없을 때 적용) 설정. 금지 사항·허용 범위·마무리 안내 문구. RAG 설정(유사도 기준값·약관 RAG 소스 상태). FAQ RAG Q&A 관리는 19_faq-rag.html로 분리. 관리자 전용. 저장 즉시 Clark AI 반영. 카드 승인 흐름 없음. 상태 개념 없음. RAG 유사도 기준값은 관리자만 변경 가능.
19_faq-rag.html FAQ RAG 관리 약관 RAG가 커버하지 못하는 Clark 서비스 전용 Q&A + 계약 무관 보험 판단 노하우 등록·관리. 목록 테이블(상태 필터·검색), 등록 패널(Q·A 필수 + 출처·⚠️유의사항 선택, 2026-07-06 4필드 확장), 상세 모달(수정·삭제·인덱스 등록). 계약 특정적 질문 등록 금지. 운영자가 Q&A 직접 등록 → 즉시 인덱스 등록(승인 프로세스 없음). 상태 2종: 초안 / 인덱스 등록됨. ⚠️ 유의사항은 LLM이 재구성하지 않고 원문 그대로 답변에 고정 부착(Policy 면책 문구와 동일 원칙).
Playbook
16_playbook-list.html Playbook 목록 전체 Playbook 카드 목록. 발화 키워드 수·CTA 버튼 수 표시. 신규 등록 버튼 텍스트: "신규 등록" (Playbook 등록 아님). table-layout:fixed 적용.
16_card-editor-playbook.html Playbook 편집기 5-card 구조: 기본정보 / 리드 전환 레이어(발화 키워드·CTA·Standalone 가이드) / 리드 스코어링(Phase 1.5 잠금) / 배정 최적화(Phase 2 잠금) / 액션. 공개범위 필드 표시 금지(내부 전용 고정). Standalone 답변 가이드 작성 참조: 13_answer-logic.html.
참조·가이드
12_coverage-code-table.html 담보코드 MYDATA 담보코드 전체 목록 조회. Rule 조건 빌더에서 참조용. 읽기 전용 참조 화면.
13_answer-logic.html AI 답변 생성 로직 Clark AI 답변 생성 전체 흐름 스펙. Case 1~4 정의, Risk-type 우선순위 공식(Step 2-B), 출력 필드 연결 범위. 체인 번호 순서: ①Concept→②Risk→③Rule→④Evidence→⑤Policy 고정.
15_aio-guide.html AIO 가이드 AI 최적화(AIO) 활용 방법. Concept·Risk-type 필드의 AIO 색인 방식 안내. AIO 대상: 고객 공개 카드(Concept·Risk-type)만. Evidence·Rule·Policy·Playbook은 AIO 색인 제외.
17_system-data-guide.html 데이터 연결 구조 Clark AI와 외부 데이터 소스(마이데이터·프롬에이지·프로파일) 연결 구조 설명. 체인 번호 순서 13_answer-logic.html과 통일.
