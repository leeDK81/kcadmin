KC 플랫폼 — 팀별 준비 체크리스트

단계별로 각 팀이 준비해야 할 것과, 팀 간에 같이 앉아서 결정해야 할 것을 정리했습니다.
세부 기술 사항은 각 팀 리드가 채워나가고, PM은 이 흐름을 기준으로 일정과 협업을 조율하세요.

⚠️ 사전준비 단계에서 가장 먼저 해야 할 것 2가지
1. 보안 솔루션 발주 (정보보호) — 서버에 설치할 보안 장비·소프트웨어는 구매 후 납품까지 4~8주 걸립니다. 프로젝트 시작과 동시에 발주해야 개발 서버 세팅 때 맞출 수 있습니다.
2. 마이데이터·약관 데이터 외부 계약 (기획) — AI 엔진이 외부 데이터를 써야 하기 때문에, 계약이 늦어지면 개발팀 전체 일정이 밀립니다.

① 사전준비

프로젝트 시작 4주 전
기획 / PO
  • 만들 것의 목적·범위 확정 (무엇을 Phase 1에 넣고 빼는지)
  • 예산 계획 — 서버·보안 솔루션·AI API 비용 포함
  • 외부 계약 착수 발주 — 마이데이터, 약관 DB 데이터 제공사
  • 준법감시인·도메인 검수자 실사용자 확보 (누가 쓸 건지 미리 확정)
디자인
  • 유사 서비스 조사 및 화면 구조 방향 초안
  • 디자인 스타일 방향 결정 (색상·폰트 등 큰 그림)
FE프론트엔드
  • 개발 기술 방향 검토 (어떤 프레임워크, 카드 연결 캔버스 구현 방식)
BE백엔드
  • 서버 구조 방향 검토
  • 외부 데이터(마이데이터·약관 DB) 연동 방식 파악
AI 개발
  • AI 모델 후보 선정 및 비용 비교
  • KC 엔진 구조 방향 결정 (어떻게 카드를 읽어 답변을 만들 것인지)
DevOps서버·인프라
  • 서버 환경 결정 — 클라우드 vs 직접 구매 발주
  • 마이데이터 연동을 위한 망 분리 요건 파악
정보보호
  • 마이데이터·금소법 보안 규제 요건 파악
  • 필요한 보안 솔루션 선정 및 발주 발주 급함 — 리드타임 4~8주
  • 보안 인증 범위 결정 (ISMS 등)
QA품질보증
  • 전체 테스트 전략 수립 및 테스트 도구 결정

같이 해야 할 것

DevOps + 정보보호
서버에 어떤 보안 솔루션을 설치할지 공동으로 결정하고 함께 발주
기획 + 정보보호
AI가 어디까지 답변할 수 있는지 금소법·마이데이터 규제 범위 확인

② 착수

킥오프 후 2~3주
기획 / PO
  • 요구사항 문서 완성 — 무엇을 만들 것인지 전 팀에 공유
  • 전체 일정·마일스톤 확정, 킥오프 미팅 주관
디자인
  • 디자인 시스템 구축 — 색상·글꼴·공통 버튼·배지 등 규격 확정
  • 핵심 화면 와이어프레임 (카드 연결 캔버스, 카드 편집기, 검수 화면)
FE
  • 개발 환경 세팅 (DevOps 개발 서버 완료 후 가능)
  • 디자인팀 규격 기반으로 공통 컴포넌트 구조 설계
BE
  • 데이터베이스 구조 설계 핵심 선행 조건 — FE·AI팀 개발 착수 전 확정 필요
  • API 명세 작성 후 FE·AI팀에 공유
AI 개발
  • KC 엔진 설계 문서 완성 — 질의 처리 흐름, 답변 생성 방식
  • 학습 데이터 수집 및 정리 시작
DevOps
  • 개발 서버 구축 FE·BE 착수 선행 조건
  • 자동 배포 파이프라인 설계·구축 시작
정보보호
  • 개인정보 처리방침 초안 작성
  • 개발팀에 보안 가이드라인 공유 (무엇을 하면 안 되는지)
QA
  • 테스트 계획 수립 및 테스트 케이스 초안 작성 시작

같이 해야 할 것

DevOps + 정보보호
개발 서버 세팅 시 보안 솔루션 함께 설치
(WAF·취약점 스캐너 등 — 발주한 것 설치)
BE + FE + AI 개발
API 명세를 함께 확정한 뒤 각자 개발 착수 — 이 합의 없이 먼저 개발하면 나중에 다 바꿔야 함

③ 진행

개발 기간 12~16주
기획 / PO
  • 2주 단위 스프린트 운영 — 무엇을 이번 주에 완성할지 조율
  • 중간에 요구사항이 바뀌면 범위·일정 영향 즉시 공유
디자인
  • 전체 26개 화면 디자인 완성 및 개발팀에 순차 전달
  • 개발팀 구현 결과 주기적으로 확인하고 피드백
FE
  • 6종 카드 편집 화면, 카드 연결 캔버스, 검수·승인 화면 개발
  • 역할에 따라 다르게 보이는 버튼·메뉴 제어
BE
  • 카드 저장·수정·조회 API, 승인 워크플로우 개발
  • 마이데이터·약관 DB 외부 데이터 연동 개발
AI 개발
  • KC 엔진 개발 — 사용자 질문 → 카드 매칭 → 답변 생성 전체 흐름
  • LLM 연동 및 면책 고지 자동 삽입, 사전 테스트 도구 개발
DevOps
  • 스테이징(검증용) 서버 구축, 배포 자동화 완성
  • 로그·모니터링 설정
정보보호
  • 스프린트마다 개발 코드 보안 점검
  • 외부로 나가는 데이터 암호화 구간 검증
QA
  • 테스트 케이스 전체 작성 완성
  • 스프린트 데모마다 참여해 검수

같이 해야 할 것

DevOps + 정보보호
스테이징 서버에 보안 솔루션 설치 및 보안 설정 적용
FE + 디자인
매주 구현된 화면을 디자인팀이 확인·피드백 (늦게 할수록 수정 공수 증가)

4~6주
기획 / PO

  • 검수자·준법감시인 UAT 참여 일정 조율
  • 오픈 전 최종 확인 목록 작성
디자인

FE
  • 발견된 버그 수정, 느린 화면 성능 개선
BE
  • API 부하 테스트 — 동시 접속자 많을 때도 버티는지
  • 버그 수정, 데이터 이전 스크립트 검증
AI 개발
  • AI 답변 품질 평가 및 튜닝
  • 예외 상황(케이스 미매칭·Fallback) 전수 점검
DevOps
  • 프로덕션(실 서비스) 서버 구축
  • 서버 장애 발생 시 복구 시나리오 테스트
정보보호
  • 외부 침투 테스트 실시 (전문 업체 진행) 용역
  • 보안 취약점 보고서 작성 및 개인정보 영향평가 완료
QA
  • 전체 기능 통합 테스트 (시나리오 기반 E2E)
  • 성능·부하 테스트, UAT 진행 및 이슈 관리

QA + AI 개발

1~2주
기획 / PO

  • 실사용자 교육 진행 (운영자·검수자·준법감시인)
  • 운영팀 인수인계 — 오픈 이후 누가 관리할지
디자인
  • 프로덕션 최종 화면 확인

BE

  • 오픈 직후 API 응답 모니터링

  • 개인정보 처리방침 게시
